hugo to zola

hugo to zola

2024-11-20 | 2mins

So am migrating to zola mostly because I don't like hugo templating system, their documentation and their naming conventions. This is a zola site I made from scratch and I tend to keep it absolute minimal syntax wise. One of the best selling feature of zola is backlinks and easy to understand documentation. Eveything I want is documented in a single page with types included. Example pages-section has every types included in a single coherent manner. This makes creating templates so much easier and neater with extra advantage of a proper naming convention. Everything is contained in page variable as mentioned in docs.

Here is an example of backlink: this is a backlink to reverse proxy and now my reverseproxy page automatically gets updated with backlink to this page. This feature is very handy and one I never thought of even though I use obsidian.
