Scrolling wm are saikoudes.

I have been sleeping on scrolling window managers. Scrolling window managers are the best. This also completely removed tmux from my workflow.

In modern day and age people open tons of program that can't be contained on a single monitor so concepts of workspace was made. This allowed extra programs to be out of view by putting them in virtual workspace which we can access later.

But this does not solves the problem. Virtual workspace gets cluttered easily. What if you are doing a project and you are opening 10 terminals, now you have to separate those terminal into 2-3 different workspace. You now need to remember which workspace you divided your project into.

I tried to solve this by using tmux. Tmux is great and you should use it! It manages a terminal into multiple panes and a single terminal can have multiple panes divided by multiple tabs. This is exactly like workspace but for terminal. But this has some problem, using tmux means some very cool features of specific terminal becomes unusable under tmux. Like search feature and scrollback writing feature of foot is very handy but I can't use it due to tmux being an overlay on top of foot. And lastly this only applies to terminal applications.

Here comes scrolling wm's. Hear me out, what if instead of virtual workspace being able to display only 1 workspace, it expanded infinitely on horizontal dimension. Welcome to scrolling window managers. A virtual workspace can have infinitely scaled horizontal dimension. So if you use 10 virtual workspace then you will have 10 infinite workspace. This means I can use workspace 1 to have all the browsing and research needs. If I need to open another browser or context I can just "scroll" right and put my application there, in the same workspace. This means I can make a mindmap that nth workspace is for certain stuff so I don't have to fragment a project into multiple workspace.

Fig: Screenshot of Niri, as you can see a single workspace can have multiple applications opened.

For now I'm using niri and hyprland with scrolling plugin. Hyprland has not caught up to scrolling like niri but I've been trying my best to make the keybinds same so I can switch between them seamlessly.

Final verdict: Fibonacci method of opening is inefficient, master layout is niche and floating window are useless. Scrolling is the future!
